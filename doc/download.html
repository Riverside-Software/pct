Statistics in 0.15

What ?

PCT 0.15 introduces a one month period during which every task will send an HTTP ping to a server.

Why ?

PCT started in 2003 because I needed a way to automate compilation for the flagship product of my company. During six years, I received bug reports, enhancements, code, documentation, and thanks (and I'm not expecting anything more) from people around the world, and from widely known companies.

However, apart from download stats, I don't have any usage statistics. Therefore, I don't know if investing some time on developing new tasks may be of any use.

When ?

PCT 0.15 will be released on September, 1st. On October 1st, PCT 0.15 will be superceded by PCT 0.16, which will be the exact copy of PCT 0.15 without the statistics code.

How ?

Every PCT task (those extending com.phenix.pct.PCT) will send an HTTP request to http://pct.rssw.eu/ping/task_name. This HTTP request is sent in a separate thread, so this won't slow down your tasks. The only informations I will have access to are your IP address, the task name and date/time.

FAQ

I'm behind a proxy, how does it work ?

If you're running Ant 1.7 (recommended !) and JDK 1.5+ (recommended too !), add -autoproxy to your command line.

Otherwise, you should use the ANT_OPTS environment variables :

  • UNIX : export ANT_OPTS="-Dhttp.proxyHost=proxy_host -Dhttp.proxyPort=proxy_port"
  • Windows : set ANT_OPTS=-Dhttp.proxyHost=proxy_host -Dhttp.proxyPort=proxy_port

Is there any risk ?

Apart from migrating to PCT 0.15, no.

Will you publish the results ?

No, I won't.

Is there something to win ?

No. However, you can guess the number of times the server will be pinged during Monday 14th. The closest answer will earn a virtual beer during Exchange Online.

I'm using my own custom version of PCT. What can I do ?

Download the source code from PCT 0.15 on this page, and merge the modifications in the constructor of com.phenix.pct.PCT class.
