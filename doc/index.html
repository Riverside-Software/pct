Overview

PCT (Progress Compilation Tools) is a set of ANT tasks, designed to compile Progress® procedures and dump/load databases.

PCT is distributed under the Apache License.

This project was born due to the lack of automated Progress® code compiler. PCT is a command-line based tool, so it can be run in batch mode, to generate nightly builds for example. However, as PCT is a set of ANT tasks, your compilation tasks can be run using a graphical frontend to ANT.

PCT was designed and written by Gilles QUERRET (g.querret at gmail.com), freelance developer for Riverside Software in France.

Collecting statistics

The forthcoming 0.15 release of PCT will introduce a two-weeks period in which every PCT task will ping a remote server. But why ?? Over years, I've put some spare time working on PCT, and got retribution in the form of bug reports, enhancements and thanks (and I don't ask for more). However, I don't know how widely PCT is used, and if it's worth spending more time developing tasks. This period will let me collect anonymous statistics on PCT usage, and decide if I need to spend more time on it.

How does it work ?

Download PCT 0.15, and execute your build files ! No change at all ! When running a task, a background thread sends an HTTP connection to a specific host (http://pct.rssw.eu), so that a log is generated server-side.

Are data anonymous ?

Almost. What's collected is your IP address, task name and date/time

What if I'm behind a proxy ?

If you're running Ant 1.7 (recommanded !) and JDK 1.5+ (recommended too !), add -autoproxy to your command line.

Otherwise, you should use the ANT_OPTS system property :

  • UNIX : export ANT_OPTS="-Dhttp.proxyHost=proxy_host -Dhttp.proxyPort=proxy_port"
  • Windows : set ANT_OPTS=-Dhttp.proxyHost=proxy_host -Dhttp.proxyPort=proxy_port
