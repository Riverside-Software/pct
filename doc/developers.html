Improving PCT

Introduction

One of the best ways to improve PCT is to report bugs and missing features. Doing this is just a matter of minutes : drop me a mail with PCT in the title, describing your problem.

Want to do it yourself ?

Changing PCT's behavior isn't hard, as long as you know Progress and Java. I'm developing PCT using Eclipse, on both Windows and Linux platforms. The basic steps to develop with PCT are :

  • Install Eclipse (3.1 or 3.2)
  • Install Subversion plugin (download here)
  • Checkout PCT from https://svn.sourceforge.net/pct/trunk/PCT
  • The project is fully configured, just modify pct.build.properties according to your config, and play with classes and procedures !!!
  • The 'Run' and 'Run external' menus should show how to run test cases, to check you didn't break anything
  • Running the 'dist' task in the buildfile will provide you a ZIP file. This is exactly the ZIP file I upload on Sourceforge myself
